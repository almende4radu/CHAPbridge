1.       CHAP MEMO Data Model Overview

CHAP MEMO Data Model aims to define an API that supports a business domain-independent and as much as possible data storage model for multiple business domains on which Artificial Intelligence modules can be defined.

This is done by implementing all data structures on top of a general purpose data storage using a storage technology-independent database called MemoNodes.

2.       MemoNodes Documentation

                                                             1.      Explanation

MemoNodes supports extensions with additional model abstraction layers for Model Driven Engineering.

It provides a clear, simple API, implemented as a REST-full and/or JSON-RPC interface.

MemoNodes is implemented as a generic storage-independent graph database. There are two implementations packaged with the core graph database: an in-memory data store and one data store implemented based on Google App Engine. This showcases the possibility to build simple, efficient, and scalable data stores managed using Platform as a Service (PaaS).

This can be extended to implement any data storage for web-based applications which need to share data. CHAP MemoNodes uses the existing Google App Engine as implementation, but it is designed to be platform-independent and can fit other implementations as well.

The main features of MemoNodes are:

●      Java library implementation using Java Beans-like objects

●      Support exporting data to JSON format for further processing by other CHAP components

●      Generic, general purpose, data storage-independent platform, based on a graph representation

●      It is an open-source, modular, open-standards supporting and easy to extend solution

●      Easy to install, to set up and to use

●      Lightweight, has a very small memory usage

 

                                                             2.      Functionality

MemoNodes displays several interesting design features specific for a temporal associative memory with grouping, filtering and querying features. It allows dynamic data structures to be formed and transformed using links and dynamic relations similar to what is happening with the neurons of a brain, that encode concepts.

Among the features of MemoNodes, we count:

●      The cells of MemoNodes memory are completely immutable Nodes: any change in value and/or Arcs will lead to a new Node creation, but with the same node ID (the old value will become discarded, but still available in the node history).

●      Arcs point to Node IDs, all nodes representing the same value (at different times) share the same Node ID.

●      Nodes can be added into multiple memory clusters, i.e. groups of nodes sharing similar attributes or grouping criteria

●      Nodes are clustered around creation time, which makes retrieval relatively fast in most cases (that is, when the most recent values are retrieved)

●      The history of one node can be retrieved by invoking method Node.history()

●      The entire graph database (MemoNodes memory) can be reset, i.e. cleared of all nodes (Tabula rasa)

●      Memories can be shared by exchanging a Node ID corresponding to a node from which the entire memory sub-graph can be visited.

●      The query Language of this graph database is a graph structure pattern matching algorithm which can itself be stored in the graph database, as a subgraph

●      The structure of the memory graph is the essence of the associative memory: you can find nodes based on some structural properties they have; this can also be used for temporal patterns

●      MemoNodes is a self-modifying storage model: its structure changes over time, allowing updates of its values

You can use MemoNodes with very little training on the use of its API, as illustrated below.

                                                             3.      Example

CHAP MemoNodes is a general purpose graph-based data store (graph database) implemented on top of the Google App Engine (GAE) cloud data store.

Reference URL[1]: https://github.com/almende/MemoNodes

A graph database stores data in a graph representation, the most generic of data structures, capable of representing any kind of data in an accessible, simple to comprehend and conceptually elegant manner. The graph contains nodes and directed arcs connecting them, to encode data attributes of elements from an arbitrary domain.

MemoNodes can express state properties, such as:

●      Values of a set of Properties at a given time. (Entity E has property P with value V at time T)

●      Existence of Relations at a given time.  (Entity E has relation R with a given entity or group of entities G?)

Such properties can be encoded in a graph-based representation as shown below.

                                                             4.      Example Usage Scenario MemoNodes

This is how you can instantiate and use Nodes and Arcs.

1.       Download the package from GitHub, either as zip package or as clone of git repository:

●      git clone https://github.com/almende/MemoNodes.git

2.       Import the project MemoNodes into Eclipse

●      Import --> General --> Existing Projects into Workspace

3.       Create a Google Web Application project that extends/uses MemoNodes. Note: Ensure you have Google Plugin installed, from Google Web Toolkit Update Site - http://dl.google.com/eclipse/plugin/3.7

●      New --> Project --> Google --> Web Application Project: fill in your project name (e.g. MemoActors) and package name (e.g. com.almende.chap.memo.memoActors)

●      In Google SDKs section check Uses Google App Engine, then Use default SDK

●      Also check box Generate project sample code

4.       In the freshly generated project MemoActors, edit properties and include project MemoNodes amongst the dependent projects. Alternative is to export the Eclipse project MemoNodes as JAR and copy it in the MemoActors/war/WEB-INF/lib folder, then add it to the list of Libraries JARs.

5.       Then edit the automatically generated servlet com.almende.chap.memo.memoActors.MemoActorsServlet.java to include com.almende.chap.memoNodes.*;

6.       This allows you to add a simple memory creation, as shown by the code below:

// create entry point, memory root node with NodeID start

MemoNode startNode = new Unode(Node.store("start"));

 

MemoNode one = new Unode(Node.storeAsChild("One", startNode).child);

MemoNode two = new Unode(Node.storeAsChild("Two", startNode).child);

MemoNode three1 = new Unode(Node.storeAsChild("Three", one).child);

three1.addChild(one); // Loop

MemoNode three2 = new Unode(Node.storeAsChild("Three", two).child);

three2.addParent(one); // multipath

This sequence creates the graph structure shown below.

Memory:

Start (ROOT)

/ \

Act Two

\ /

Three

 

Note: A Unode is a reference to a Node. MemoNode is the actual (implementation-specific) content of the Node. Node is a bean class wrapping all the attributes and methods of a generic graph node.

7.       Querying can be done by the following code:

MemoNode algorithm = new Unode(Node.store("algo"));

Node.storeAsChild("Pattern", algorithm).child);

MemoNode patFive = new Unode(

MemoNode PreStart = new Unode(

Node.storeAsChild("equal;start", preAmble).child);

MemoNode PreOne = new Unode(

Node.storeAsChild("equal;One", PreStart).child);

Note: MemoNodes supports a pattern querying and matching language. The syntax of this language is: <OperandNode> <OperatorNode; OperandNode>, where <OperandNode> is a label depicting the name of a node, and <OperatorNode> is an operator prefix such as equal,.

MemoNodes API contains the following classes:

Class Node {

    public static Node store(String value);

    public Node update(String value);

    public static Node find(String id);
    public String valueAt(Date timestamp);
    public ArrayList<Node> history();  //Not implemented yet!
    public String getId();
    public String getValue();
    public Date getTimestamp();

    public Arc addParent(Node node);
    public Arc addChild(Node node);

    public ArrayList<Node> getChildren();
    public ArrayList<Node> getParents();

}

    public Node child = null;

                                                             7.      Using MemoNodes for a specific application domain - MemoActors

MemoActors is a general purpose (domain-independent) memory which allows implementation of an agent memory based on a MemoNodes implementation.

Reference URL[2]: https://dev.almende.com/svn/bridge/GAE_GAME3

MemoActors contains two main classes which extend/reuse the Nodes functionality from MemoNodes: Actors and Interactions.

An Interaction is a given state and the event that led to that state. Occurrence of an Event which leads to a state change is represented as by an Interaction node that links all Actors (i.e. parties) involved in that interaction.

One of the goals of Memo is to have emergent clusters of Actors, clustering on some shared aspect. Many aspects can be used to cluster actors together, among them Geo-Location, Temporal co-occurrence or causality, Domain-specific relations (such as, business meetings), Communication Medium or Protocol used, same property values (e.g. same medical condition), etc.

public class ActorNode extends MemoNode implements Serializable, IActor {

private Long id;

private String actorName;

 

private long timestamp;

public static final String UNDEFINED = "UNDEFINED";

 

HashMap<String,Property> properties = new HashMap<String,Property>();

HashMap<String,Actor> parents = new HashMap<String,Actor>();

HashMap<String,Actor> children = new HashMap<String,Actor>();

HashMap<String,Interaction> interactions = new HashMap<String,Interaction>();

 

public ActorNode(Node myNode) {

super(myNode);

}

public ActorNode(String actorName) {

this("A"+System.currentTimeMillis(), actorName, System.currentTimeMillis());

}

public ActorNode(String id, String name, long timeStamp) {

super(new Node(id));

properties = new HashMap<String, Property>();

this.addActorProperty("name", new Property("name", name));

this.setLastUpdate(timeStamp);

}

public ActorNode(String actorName2, ArrayList<String> actorPropertyNames) {

super(new Node(actorName2));

PropertyNode pn = null;

for (String propertyName : actorPropertyNames) {

addProperty(propertyName, UNDEFINED);

result = result.addChild((MemoNode) propertyName));

}

}

}
